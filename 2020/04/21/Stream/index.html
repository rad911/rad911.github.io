Stream
发布于 2020-04-21   更新于 2020-04-24 Java 8

Java 8 中的 Stream 是对集合对象功能的增强，它专注于对集合对象进行各种非常便利、高效的聚合操作或者大批量数据操作。Stream API（java.util.stream.*）提供了一种高效且易于使用的处理数据的方式。使用Stream API可以执行非常复杂的查找、过滤和过滤数据等操作。使用Stream API对集合数据进行操作，就类似于使用SQL执行的数据库查看找，非常方便。也可以使用Stream API来并行执行操作。

什么是流

“集合讲的是数据，流讲的是计算！”

注意点：

  1. Stream自己并不会存储数据
  2. Stream不会改变源对象。在完成相应的计算，会返回一个持有结果的新Stream。
  3. Stream的操作是延迟执行的。这意味着会等到需要结果的时候才去执行。

Stream的三个操作步骤

创建Stream -> 中间操作 -> 终止操作

1.创建Stream

使用数据源（如：集合、数组），获取一个流

2.中间操作

多个中间操作可以连接起来形成一个流水线，除非流水线上触发终止操作，否则中间操作不会执行任何的处理，而在终止操作时一次性全部处理，称为“惰性求值”。

2.1 筛选与切片

方法 描述
filter (Predicate p) 接收Lambda，从流中排除某些元素。Predicate接口的唯一抽象的方法为boolean test(T t);。
distinct() 筛选，通过流所生成元素的hashCode()和equals方法去除重复元素。
limit(long maxSize) 截断流，使其元素不超过给定数量。
skip(long n) 跳过元素，返回一个扔掉了前n个的元素的流。若流中的元素不足n个，则返回一个空流。与limit(n)互补。

2.2 映射

方法 描述
map(Function f) 接收一个函数作为参数，该函数会被应用到每个元素上，并将其映射成一个新的元素。Function接口的唯一抽象方法为R apply(T t);。
mapToDouble(ToDoubleFunction f) 接收一个函数作为参数，该函数会被应用到一个每个元素上，产生一个新的DoubleStream对象ToDoubleFunction接口唯一的抽象方法为double applyAsDouble(T value);。
mapToInt(ToIntFunction f) 接收一个函数作为参数，该函数会被应用到每个元素上，产生一个新的IntStream对象。ToIntFunction接口唯一的抽象方法为int applyAsInt(T value);。
mapToLong(ToLongFunction f) 接收一个函数作为参数，该函数会被应用到每个元素上，产生一个新的LongStream对象。ToLongFunction接口唯一的抽象方法为long applyAsLong(T value);。
flatMap(Function f) 接收一个函数作为参数，将流中的每个值都换成另一个流，然后把所有的流连接成一个流。Function接口的唯一抽象方法为R apply(T t);。

2.3 排序

方法 描述
sorted() 产生一个新流，其中按照自然顺序排序。
sorted(Comparator comp) 产生一个新流，其中按比较器顺序排序。

3.终止操作

终止操作会从流的流水线生成结果。其结果可以是任何不是流的值，例如：List、Integer，甚至是void。

3.1 匹配

返回boolean值

方法 描述
allMatch(Predicate p) 检查是否匹配所有的元素。Predicate接口的唯一抽象的方法为boolean test(T t);。
anyMatch(Predicate p) 检查是否至少匹配一个元素。Predicate接口的唯一抽象的方法为boolean test(T t);。
noneMatch(Predicate p) 检查是否没有匹配所有的元素。Predicate接口的唯一抽象的方法为boolean test(T t);。

3.2 查找

方法 描述
findFirst() 返回第一个元素。
findAny() 返回当前流中的任意元素。
count() 返回流中的元素总数。
max(Comparator c) 返回流中的最大值。
min(Comparator c) 返回流中的最小值。

3.3 迭代

返回值为空

方法 描述
forEach(Consumer c) 内部迭代。

根据打印顺序显示，forEach 方法开启了一个新的线程。在这个线程开启后，外界操作对其中的集合将没有任何影响。

3.4 规约

方法 描述
reduce(BinaryOperator b) 可以将流中元素反复结合起来，得到一个值，返回Optional。BinaryOperator（二元运算符）继承于BiFunction，唯一的抽象方法为R apply(T t, U u);。
reduce(T iden, BinaryOperator b) 可以将流中元素反复结合起来，得到一个值。iden代表的是初始值。

3.5 收集

方法 描述
collect(Collector c) 将流转换为其他的形式。接收一个Collector接口的实现，用于给

Collectors类提供了很多静态方法，可以更方便的创建常见的收集器（Collector）实例。

3.5.1将流中元素规约和汇总为一个值
3.5.1.1 查找流中的最大值和最小值

可以使用Collectors.maxBy和Collectors.minBy，来计算流中的最大值和最小值。这两个收集器接收是一个Comparator参数来比较流中的数据。

3.5.1.2 汇总

（1）求和

Collectors类专门为汇总提供了一个工厂方法：Collectors.summingInt。它可以接受一个把对象映射为求和所需int的函数，并返回一个收集器；该收集器在传递给普通的collect方法后即执行我们所需要的汇总操作。

Collectors.summingLong和Collectors.summingDouble方法的作用完全一样，可以用来求和字段为long和double的情况。

（2）计算平均数

与求和类似的，Collectors类还提供了averagingInt，averagingDouble，averagingLong用来计算平均值。

（3）统计数据对象

如果需要统计多个属性，stream为我们提供了summarizingInt/Double/Long等方法，方便对对象的int、double、long等类型的属性进行统计。这个三个方法会返回元素该属性的统计数据的对象，分别对应IntSummaryStatistics、DoubleSummaryStatistics、LongSummaryStatistics，包含了属性的个数，总和，最大值，最小值，平均值等5个指标。

3.5.1.3 连接字符串

joining工厂方法返回的收集器会把流中的每个对象应用toString方法得到的所有字符串连接成一个字符串。joining在内部使用了StringBuilder来把生成的字符串逐个追加起来。joining方法方法还有一个带参数的重载版本，可以接受元素之间的分解符，不带参数则是没有分隔符（是没有分割符而不是用空格连接）。
